PostGIS Geo Query 
-- Find reports within radius (metres) 
SELECT * FROM reports 
WHERE ST_DWithin( 
  location::geography,   ST_MakePoint(:lng, :lat)::geography, 
  :radiusMetres 
) 
ORDER BY ST_Distance(location::geography, ST_MakePoint(:lng, :lat)::geography); 
 
⚠ Use ::geography cast for accurate metre-based distance. Without it, ST_DWithin uses degrees. 
 
Duplicate / Support Logic

• If match found → call POST /reports/{existingId}/support instead of creating new report

(none) ASSIGNED Admin approves bid

Phase 5 — Priority Scoring Engine 
 
Priority Score Formula 
// PriorityService.calculateScore(report) 
int supports     = report.getSupportCount(); 
int severity     = categoryToSeverity(report.getCategory()); // 1-5 
long daysOpen    = ChronoUnit.DAYS.between(report.getCreatedAt(), Instant.now()); 
int ageFactor    = (int) Math.min(daysOpen * 2, 20);  // cap at 20 
 
int score = (supports * 2) + (severity * 5) + ageFactor; 
 
// Map to priority level 
// < 10  → LOW 
// 10-24 → MEDIUM 
// 25-39 → HIGH 
// 40+   → CRITICAL 
 
When Priority is Recalculated 
• New support added to report 
• Scheduled daily job recalculates age factor for all open reports

JWT Validation Filter 
// GatewayFilter that validates JWT before forwarding 
public class JwtAuthFilter implements GatewayFilter { 
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) 
{ 
        String token = extractToken(exchange.getRequest()); 
        if (!jwtUtil.isValid(token)) { 
            exchange.getResponse().setStatusCode(HttpStatus.UNAUTHORIZED); 
            return exchange.getResponse().setComplete(); 
        } 
        return chain.filter(exchange); 
    } 
}
